# Content Ops Architecture

## Core stack
- n8n: orchestration and agent workflows
- Supabase: memory, state and operational data
- Cloudflare Workers: approval panel and lightweight API
- Buffer: social publishing
- GitHub: versioning, prompts and documentation

## Initial flow
1. Topic candidates are created
2. Briefs are generated
3. Assets are generated by platform
4. Assets are reviewed
5. Human approves or rejects
6. Approved assets are published through Buffer
7. Metrics are collected later
